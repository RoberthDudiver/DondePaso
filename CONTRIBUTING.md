# Contributing to DondePaso

Thanks for helping shape DondePaso.

This project is meant to grow with the community around a simple idea: revealing how much of our real-world surroundings we actually explore.

## Where to post what

- Use `Discussions` for ideas, questions, product feedback, and open conversation.
- Use `Issues` for confirmed bugs, focused feature requests, and actionable work.
- Use `Pull Requests` when you already have a concrete code or documentation change.

## Before opening a PR

- Keep changes focused.
- Explain the problem and the reason for the change.
- Include screenshots or short videos when the UI changes.
- Mention platform impact when relevant, especially Android background behavior and permissions.

## Good contribution areas

- UX and onboarding clarity
- Privacy-first wording
- Background tracking stability
- Battery optimization
- Localization improvements
- Community and documentation

## Product principles

- Privacy first
- Local-first by default
- Clear, simple UI
- Honest communication about permissions and limitations
- Exploration over surveillance language

## Reporting bugs

When possible, include:

- device model
- Android or iOS version
- app version
- what you expected
- what actually happened
- steps to reproduce

## Feature ideas

If you suggest a feature, it helps to explain:

- the user problem
- the expected behavior
- why it fits DondePaso
- any privacy or battery tradeoffs
